webots 的模型有沒有辦法用程式方法建立? 有沒有辦法直接從 CAD 套件中轉出 webots 的場景模型？

✅1. Webots 是否能用程式方法建立模型？

可以，有兩種主要方式：

#(A) 直接產生 .wbt 或 .proto 檔（最常用的方法）

      Webots 的世界檔案 *.wbt 與模型 *.proto 都是 純文字格式（基於 JSON-Like 的 Scene Tree 語法）。

你可以用任何語言寫程式（Python / C++ / JS / 等）產生

下面是程式範例:

Robot {
  translation 0 0 0
  children [
    Shape {
      geometry Box {
        size 1 1 1
      }
    }
  ]
}

丨只要輸出成 .wbt 或 .proto 就能直接被 Webots 載入。

  • 用途：大量產生物件、程序化地形、建立大型環境、模型版本控制等。

#(B) 在 Webots 中用 Supervisor API 動態建立節點

      Supervisor 程式可在 simulation runtime 插入整個 Node。

以下是python程式範例:

from controller import Supervisor
super = Supervisor()

root = super.getRoot()
children = root.getField("children")

node_string = """
DEF Box Shape {
  geometry Box {
    size 1 1 1
  }
}
"""

children.importMFNodeFromString(node_string)

丨可以動態加入：

  • Shape、Solid、Robot
  • 完整的 Prototype（PROTO）

但不適合建立非常複雜的場景（效率較差）。

✅ 2. 能不能從 CAD 套件直接轉出 Webots 場景？

     沒有官方支援「CAD → Webots」的直接匯出器。

但可以透過以下方式完成：

(A) CAD → STL/OBJ → Webots（最常用）

    大部分 CAD（SolidWorks、Fusion360、Creo、Inventor、Rhino…）都能輸出：

  • .stl
  • .obj

Webots 可直接把這些模型載入成 Shape。

    程式範例:

Shape {

  geometry IndexedFaceSet {

    coord Coordinate { point [...] }

  }

}

或是使用:

Mesh {

  url "model.stl"

}

丨適用於靜態物件或機械零件。

(B) CAD → URDF/SDF → Webots

    如果你的模型是機器（多關節）：

1️⃣ 用 CAD（例如 SolidWorks + SW2URDF plugin）匯出 URDF

2️⃣ Webots 可直接 import URDF → 自動轉成 PROTO 機器

丨適用於帶關節、連桿、馬達的模型。

(C) CAD → Blender → Webots

    Blender 可作為中間站：

  • 匯入 STEP/IGES
  • 修復 mesh（decimate、合併、法向調整）
  • 匯出為 OBJ 或 STL → Webots

丨適用於複雜模型與材質處理。
